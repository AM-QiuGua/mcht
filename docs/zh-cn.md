# 🎮 Minecraft Help Tools（MC指令&ID全能工具箱）
一款基于Python + PySide6开发的我的世界可视化指令生成工具，服务端OP、地图创作者专用，内置完整指令、物品、方块、药水、附魔数据库，一键拼接复制指令，无需记忆复杂语法。

最新版本：v1.0.0 （2026.7.5）

## ✨ 核心功能
1. **五大分类完整数据库**
    - 服务器指令：关服、OP、传送、给予物品、游戏规则、效果、附魔等全套后台指令
    - 物品ID：钻石装备、下界合金、食物、工具、道具全收录（覆盖1.0–1.21）
    - 方块ID：建筑、红石、功能性方块、新版本特色方块
    - 药水效果ID：增益/减益buff完整清单，支持自定义时长、等级
    - 附魔ID：装备全附魔，支持手持/盔甲架/实体附魔目标切换
2. **多维度筛选检索**
    - 关键词搜索：名称、ID、说明模糊匹配
    - 类型筛选：单独查看指令/物品/方块/效果/附魔
    - 版本过滤：筛选1.0 ~ 1.21对应版本内容，避免版本不兼容指令
3. **可视化指令自动拼接**
    内置参数面板，修改参数实时预览完整指令：
    - 目标选择器：@p / @a / @s / @e / @r / 自定义玩家名
    - 数量/等级、药水时长、附魔目标一键调整
    - 自动生成 `give`、`effect`、`enchant`、`fill` 等标准格式指令
4. **一键复制**
    - 复制核心ID/原生指令
    - 复制拼接完成的完整可执行指令，直接粘贴至服务端控制台/命令方块
5. **轻量化UI界面**
    自适应窗口、简约分层布局、代码高亮预览、清晰条目统计，无多余冗余功能。

## 📁 项目目录结构
```
MCHT/
├── Minecraft_Help_Tools.py   # 主程序入口、Qt界面逻辑
├── data.json                 # 全量数据库（指令/物品/方块/效果/附魔）
```
> 运行要求：`Minecraft_Help_Tools.py` 与 `data.json` 必须放在同一文件夹，否则程序会提示文件缺失并退出。

## 📖 软件操作使用说明
1. **筛选区域**
    - 关键词：输入物品名、指令名、ID快速检索
    - 类型：切换查看指令/物品/方块/药水效果/附魔

3. **右侧详情面板**
    - 详情说明：展示物品/指令介绍、支持版本
    - 核心内容：原生ID/原始指令，可单独复制
    - 指令参数拼接区：修改玩家、数量、时长、附魔目标，预览实时更新
    - 指令预览：生成完整可直接执行的MC指令
4. **复制按钮**
    - 复制核心内容：仅复制ID/原生指令模板
    - 复制完整指令：复制拼接好、替换完目标参数的成品指令

## 📊 数据库内容说明（data.json）
JSON分为5大数组存储全部游戏数据，每条数据统一字段规范：
- `name`：中文显示名称
- `cmd` / `id`：游戏原生指令/资源ID
- `desc`：功能详细说明
- `version`：最低支持MC版本

1. **commands 服务器指令**
    关服、存档、OP、传送、给予物品、游戏难度、时间、天气、游戏规则、清除实体、附魔、药水、填充方块等控制台常用指令。
2. **items 物品库**
    矿石、装备、食物、工具、道具、1.16下界合金、1.19幽匿、1.21试炼道具全覆盖。
3. **blocks 方块库**
    建筑方块、红石元件、功能性方块、下界/末地特色方块、新版本试炼方块。
4. **effects 药水效果库**
    全部增益、减益状态效果，支持自定义时长、等级生成effect指令。
5. **enchantments 附魔库**
    武器、工具、盔甲、三叉戟、十字弩全附魔，区分手持/盔甲架附魔场景。

## ⚠️ 常见问题与故障排查
1. **启动提示未找到data.json**
    解决方案：将`data.json`与`Minecraft_Help_Tools.py`放在同一个文件夹，不要嵌套多层目录。
2. **复制按钮提示复制失败**
    原因：剪贴板被其他软件占用，关闭录屏、剪贴增强工具后重试。
3. **筛选后列表为空**
    - 关键词输入错误；
    - 版本筛选设置过高，当前条目最低版本大于筛选版本；
    - 分类筛选不匹配条目类型。
4. **生成指令不符合预期**
    检查参数面板自定义玩家名、等级、时长是否为空，程序会自动填充默认值兜底。
