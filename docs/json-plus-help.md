## 1. 文档概述
MCHT 工具全部内置素材、指令、游戏数据统一由项目根目录 `data.json` 驱动。
程序启动时自动读取此文件，解析后渲染软件内全部列表界面；**无需修改程序代码，仅编辑 JSON 即可自定义所有列表展示内容**。

原生内置5大分类数组，对应软件5个功能列表：
- `commands`：服务器控制台指令列表
- `items`：游戏物品ID资料库
- `blocks`：游戏方块ID资料库
- `effects`：药水状态效果资料库
- `enchantments`：装备附魔资料库

所有分类数据遵循统一设计逻辑，修改、新增、拓展规则通用。

## 2. data.json 整体结构
```json
{
  "commands": [
    {"name": "xxx", "cmd": "xxx", "desc": "xxx", "version": "1.0"}
  ],
  "items": [
    {"name": "xxx", "id": "xxx", "desc": "xxx", "version": "1.0"}
  ],
  "blocks": [
    {"name": "xxx", "id": "xxx", "desc": "xxx", "version": "1.0"}
  ],
  "effects": [
    {"name": "xxx", "id": "xxx", "desc": "xxx", "version": "1.0"}
  ],
  "enchantments": [
    {"name": "xxx", "id": "xxx", "desc": "xxx", "version": "1.0"}
  ]
}
```
结构说明：
1. 顶层键名 = 功能分类名称，程序根据键名生成对应标签页；
2. 每个键对应一个数组 `[]`，数组内每一个 `{}` 对象 = 列表中单独一行；
3. 数组内对象从上至下顺序 = 软件列表从上到下显示顺序。

## 3. 各分类条目字段完整释义
### 3.1 commands 指令分类专属结构
```json
{
  "name": "stop 安全关服",
  "cmd": "stop",
  "desc": "保存全部区块与玩家数据后正常关闭服务器，防止存档损坏，下线前必须使用。",
  "version": "1.0"
}
```
| 字段 | 作用 | 自定义说明 & UI显示逻辑 |
|------|------|------------------------|
| `name` | 列表主展示标题 | 软件列表第一列直接展示，支持中文、符号、备注；可自由修改名称方便区分用途，如 `tp 传送玩家(相对坐标)` |
| `cmd` | 真实执行指令 | 点击条目一键复制/发送的服务端指令文本，可修改玩家名、数值、坐标参数；游戏原版指令格式不可写错 |
| `desc` | 详情说明 | 鼠标悬浮、点击详情弹窗展示；支持JSON转义换行 `\n` 分段，可增加警告、使用格式说明 |
| `version` | 版本标签 | 用于软件版本筛选器；填写对应MC版本，多版本兼容可写 `1.8~1.21`，通用内容统一填 `1.0` |

### 3.2 items / blocks / effects / enchantments 通用结构
```json
{
  "name": "钻石",
  "id": "diamond",
  "desc": "稀有矿物，制作顶级工具与装备的核心材料。",
  "version": "1.0"
}
```
| 字段 | 作用 | 自定义说明 & UI显示逻辑 |
|------|------|------------------------|
| `name` | 列表展示名称 | 界面列表主标题，自定义友好中文名称，方便玩家识别 |
| `id` | 游戏原版标准ID | 游戏底层识别标识，仅允许填写MC原版/模组真实ID，不可随意编造 |
| `desc` | 详情描述 | 悬浮弹窗展示详细用途、特性、获取方式 |
| `version` | 适配版本 | 版本筛选功能依赖此字段，控制列表过滤显示 |

## 4. 基础操作：修改现有列表内容
### 示例需求
修改 `commands` 内 `kill 清除实体` 条目，修改展示名称、补充风险提示描述
原数据：
```json
{"name": "kill 清除实体", "cmd": "kill @e[type=!player]", "desc": "清除除玩家外的所有生物与掉落物，慎用。", "version": "1.0"}
```
自定义修改后：
```json
{
  "name": "kill 一键清空世界实体(谨慎)",
  "cmd": "kill @e[type=!player]",
  "desc": "⚠高危指令\n清除除玩家外全部生物、掉落物、载具\n执行前建议先用save-all存档，避免物资丢失",
  "version": "1.0"
}
```
修改要点：
1. 修改 `name` 直接改变软件列表显示文字；
2. `desc` 内使用 `\n` 实现详情文本换行；
3. `version` 按需修改，筛选器会同步生效。

## 5. 基础操作：新增自定义列表条目
以 `commands` 新增自定义指令为例：
1. 在 `commands` 数组末尾 `]` 前添加逗号，粘贴完整对象；
2. 完整示例新增自定义指令：
```json
{
  "name": "weather thunder 全天雷暴",
  "cmd": "weather thunder 999999",
  "desc": "长时间锁定雷暴天气，可配合引雷三叉戟使用",
  "version": "1.0"
}
```
新增规则：
- 多条目之间必须用英文逗号 `,` 分隔；
- 字段不可缺失，`name/cmd/desc/version` 四键缺一不可，否则程序读取报错；
- 字段值必须使用英文双引号 `"` 包裹。

## 6. 进阶操作：调整列表排序、删除条目
1. **调整排序**：直接剪切整条 `{}` 对象，在数组内上下移动位置，软件列表同步变更顺序；
2. **删除条目**：直接移除完整 `{}` 对象，同时清理多余逗号，避免JSON格式报错。

## 7. 进阶拓展：新增自定义顶级分类
如果你需要新增独立列表（例如模组指令、领地插件指令），可在JSON顶层新增键值数组。
示例新增 `plugin_cmds` 插件指令分类：
```json
{
  "commands": [...],
  "items": [...],
  "plugin_cmds": [
    {
      "name": "res create 创建领地",
      "cmd": "res create 1",
      "desc": "创建1号私人领地，保护建筑不被破坏",
      "version": "1.0"
    }
  ]
}
```
程序会自动识别新顶层键，生成独立列表页面，字段结构与 `commands` 完全一致。

## 8. UI 列表渲染显示规则
1. 列表主视图：仅展示 `name` 字段，作为每行可见标题；
2. 右键/详情弹窗：完整展示 `name` + `cmd/id` + `desc` + 适配版本；
3. 版本筛选逻辑：选择对应版本后，仅展示 `version` 字段包含匹配文本的条目；
4. 复制功能：点击条目自动复制 `cmd` / `id` 字段到剪贴板；
5. 列表排序：默认遵循JSON数组书写顺序，暂不支持程序内自动排序。

## 9. JSON 编写规范与自定义限制
### 允许自定义
1. 所有字段内文本内容自由修改、添加符号、换行；
2. 自由增删条目、调整顺序；
3. 顶层新增自定义分类数组；
4. `version` 可自定义多版本标识，如 `1.16~1.21`、`mod1.0`。

### 禁止修改（会导致程序读取失败）
1. 不可修改固定字段名：`name` / `cmd` / `id` / `desc` / `version`；
2. 不可使用中文逗号、中文引号、中文顿号；
3. 不能删除条目内任意一个必填字段；
4. 字符串内换行必须使用转义符 `\n`，不能直接回车换行。

## 10. 常见报错、格式问题修复
1. **软件打开空白列表/列表不加载**
   - 原因：JSON语法错误（多余逗号、缺失引号、字段丢失）
   - 修复：使用在线JSON校验工具检查格式，删除数组末尾多余逗号；
2. **详情文本换行失效**
   - 原因：直接回车换行，未写 `\n`；
   - 修复：段落间手动添加 `\n`；
3. **版本筛选后内容全部消失**
   - 原因：条目 `version` 填写版本与筛选选项不匹配；
   - 修复：统一填写对应版本号，通用内容填 `1.0`；
4. **复制指令/ID为空**
   - 原因：误删 `cmd` / `id` 字段；
   - 修复：补全对应字段，填写有效内容。
